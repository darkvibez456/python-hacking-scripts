# Contributing to python-hacking-scripts

We welcome contributions to this project! To ensure a smooth and collaborative process, please follow these guidelines.

## How to Contribute

1.  **Fork the repository:** Start by forking the `python-hacking-scripts` repository to your GitHub account.
2.  **Clone your fork:** Clone your forked repository to your local machine:
    ```bash
    git clone https://github.com/darkvibez456/python-hacking-scripts.git
    cd python-hacking-scripts
    ```
3.  **Create a new branch:** Create a new branch for your feature or bug fix:
    ```bash
    git checkout -b feature/your-feature-name
    ```
4.  **Make your changes:** Implement your changes, ensuring they adhere to the project's coding standards.
5.  **Test your changes:** Before submitting, thoroughly test your changes to ensure they work as expected and do not introduce new issues.
6.  **Commit your changes:** Write clear and concise commit messages.
    ```bash
    git commit -m "feat: Add new script for X"
    ```
7.  **Push to your fork:** Push your changes to your forked repository:
    ```bash
    git push origin feature/your-feature-name
    ```
8.  **Create a Pull Request:** Open a pull request from your fork to the `main` branch of the original `python-hacking-scripts` repository. Provide a detailed description of your changes.

## Coding Standards

-   Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) for Python code style.
-   Include docstrings for all functions and classes.
-   Add comments where necessary to explain complex logic.

## Reporting Bugs

If you find a bug, please open an issue on the GitHub repository with a clear description of the problem, steps to reproduce it, and expected behavior.

## Feature Requests

For new feature ideas, please open an issue to discuss it before starting any development.
